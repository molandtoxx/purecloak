# PureCloak 用户使用说明

> 基于 Chromium 的双模式工作空间级身份隔离浏览器 — 一个浏览器，双面灵魂。

---

## 一、产品概述

PureCloak 是一款基于 Chromium 深度二次开发的专业浏览器，核心能力在于**工作空间级身份隔离**。

不同于传统的多配置文件方案，PureCloak 将每个工作空间（Workspace）作为一个完全独立的浏览器实例来运行——拥有独立的 `--user-data-dir`、独立代理、独立指纹参数、独立 Chromium 子进程，通过 CDP 协议与主进程通信。这使得不同工作空间之间的 Cookie、LocalStorage、缓存等所有存储实现**物理级隔离**，从根本上防止账号关联。

### 适用场景

| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 日常上网 / 个人浏览 | 普通工作空间 | 原生 Chromium 体验，零开销 |
| 多账号电商运营 (Amazon/eBay/Shopify) | 指纹工作空间 | 每账号独立代理+独立指纹+独立存储 |
| 社交媒体矩阵管理 | 指纹工作空间 | 每个 Profile 对应一个独立身份 |
| 广告投放验证 | 指纹工作空间 | 不同 IP 和指纹环境查看广告展示 |
| 自动化测试 (Playwright/Puppeteer) | 两种模式均可 | 通过 CDP 协议精确控制 |

## 二、核心特性

### 🔒 进程级隔离
- 每个工作空间运行在独立 Chromium 子进程中
- 拥有独立的 `--user-data-dir`
- 存储完全物理隔离（Cookie、LocalStorage、IndexedDB、Cache）

### 🛡️ 代理隧道
- 支持 SOCKS5 / HTTP / HTTPS 代理
- 通过 `--proxy-server` 启动参数注入
- 支持代理账号密码认证
- 每个工作空间可绑定独立代理

### 🧬 指纹伪装（反侦测）
- **`navigator.webdriver` 强制隐藏** — 通过 CDP `Page.addScriptToEvaluateOnNewDocument` 在页面加载前注入
- **WebRTC 强制策略** — `--force-webrtc-ip-handling-policy=disable_non_proxied_udp`
- **User-Agent 自定义** — 可配置任意 UA 字符串
- **屏幕分辨率** — 自定义宽度/高度/色彩深度
- **时区与时区偏移** — 自定义 `Intl.DateTimeFormat` 行为
- **语言环境** — 自定义 `navigator.languages` 和 `Accept-Language`
- **硬件并发数** — 自定义 `navigator.hardwareConcurrency`
- **平台标识** — 自定义 `navigator.platform`
- **色彩方案** — 自定义 `prefers-color-scheme`
- **Canvas 指纹混淆** — 通过 Anti-Detection Engine 注入确定性噪声
- **WebGL 指纹混淆** — 修改 VENDOR/RENDERER 字符串
- **AudioContext 混淆** — 在音频处理链中注入微小偏移
- **字体指纹防护** — 字体列表混淆
- **Humanize 行为模拟** — 生成模拟人类操作的 JavaScript（鼠标移动、滚动、输入时间模式）

### 🔌 CDP 远程控制
- 每个工作空间暴露独立 CDP 端口
- 兼容 Playwright / Selenium / Puppeteer
- 支持标签页级精确控制

### 🖥️ 图形化管理界面（WebUI）
- 内置 `purecloak://purecloak/` 管理页面
- 支持工作空间的创建、编辑、启动、停止、删除
- **仪表盘统计**：实时显示 Total / Running / Fingerprint / Normal 工作空间数量
- **搜索过滤**：按名称或标签搜索工作空间
- **批量操作**：多选工作空间，批量启动/停止/删除
- **标签系统**：为工作空间添加标签，便于分类管理
- **模板系统**：将当前工作空间列表保存为模板，一键加载
- **指纹预览**：实时预览指纹配置摘要
- **代理测试**：一键测试代理连接可用性
- **键盘快捷键**：`Ctrl+N` 新建、`Ctrl+F` 搜索、`Esc` 关闭对话框、`Delete` 批量删除

### 🌐 中文本地化
- 完整的中文界面
- 所有 UI 字符串支持多语言扩展

### 🔄 看护进程 (Watchdog)
- 子进程退出自动检测
- 状态实时同步到 WebUI
- 崩溃自动标记

## 四、安装与构建

### 4.1 克隆仓库

```bash
git clone https://github.com/molandtoxx/purecloak.git
cd purecloak
```

### 4.2 一键构建

```bash
# Linux (x86_64)
bash scripts/build-linux.sh
```

脚本会自动完成：
1. 安装 Chromium 构建依赖
2. 同步 Chromium 源码（gclient sync）
3. 配置 GN 构建参数
4. 编译 PureCloak

### 4.3 手动构建

```bash
# 1. 安装构建依赖
sudo ./chromium_src/build/install-build-deps.sh

# 2. 同步源码
sed -i 's/"managed": False/"managed": True/' .gclient
gclient sync --shallow --nohooks
sed -i 's/"managed": True/"managed": False/' .gclient
git checkout -- chromium_src/chrome/app/chromium_strings.grd

# 3. 配置 GN
gn gen out/purecloak --args='
  is_debug=false
  is_purecloak=true
  is_component_build=false
  symbol_level=0
  blink_symbol_level=0
  optimize_for_size=true
  enable_nacl=false
  use_sysroot=true
'

# 4. 编译（首次约 2-4 小时）
autoninja -C out/purecloak chrome

# 5. 运行
./out/purecloak/chrome
```

### 4.4 构建选项

| 参数 | 说明 |
|------|------|
| `--debug` | Debug 构建（默认 release） |
| `--out DIR` | 输出目录 |
| `--skip-sync` | 跳过 gclient sync |
| `--skip-tests` | 跳过测试编译 |
| `--jobs N` | 并行编译任务数 |

## 五、快速开始

### 5.1 启动 PureCloak

```bash
# 直接启动
./out/purecloak/chrome

# 或通过构建脚本
bash scripts/build-linux.sh --skip-sync
```

启动后，浏览器会自动打开 `purecloak://purecloak/` 管理页面。

### 5.2 创建普通工作空间

1. 在管理页面点击 **"+ New Workspace"** 按钮
2. 输入工作空间名称（如 "日常浏览"）
3. 类型选择 **"Normal"**
4. 点击 **"Create"**
5. 创建成功后，工作空间卡片出现在列表中
6. 点击 **"Launch"** 启动

### 5.3 创建指纹工作空间

1. 点击 **"+ New Workspace"** 按钮
2. 输入工作空间名称（如 "亚马逊运营"）
3. 类型选择 **"Fingerprint"**
4. **展开 "Fingerprint Settings"** 配置指纹参数：

   | 参数 | 说明 | 示例 |
   |------|------|------|
   | User Agent | 自定义 UA 字符串 | `Mozilla/5.0 (Windows NT 10.0; Win64; x64) ...` |
   | Screen Width | 屏幕宽度 | `1920` |
   | Screen Height | 屏幕高度 | `1080` |
   | Hardware Concurrency | CPU 核心数 | `8` |
   | Platform | 操作系统平台 | `windows` / `macos` / `linux` |
   | Color Scheme | 色彩方案 | `light` / `dark` / `no-preference` |
   | Timezone | 时区 | `America/New_York` |
   | Locale | 语言环境 | `en-US` |
   | Proxy | 代理地址 | `socks5://user:pass@127.0.0.1:1080` |
   | GeoIP | 启用 GeoIP 匹配 | 根据代理 IP 自动匹配时区和语言 |

5. 可选：添加**标签**（用逗号分隔，如 `美国站, 亚马逊`）
6. 点击 **"Create"**
7. 启动工作空间后，独立子进程会在 CDP 端口（默认偏移 9333）监听

### 5.4 连接已启动的工作空间

```python
# Playwright 示例
from playwright.async_api import async_playwright

async with async_playwright() as p:
    # 通过 CDP 连接到指纹工作空间
    browser = await p.chromium.connect_over_cdp("http://127.0.0.1:9333")
    page = await browser.new_page()
    await page.goto("https://example.com")
    print(await page.title())
    await browser.close()
```

```javascript
// Puppeteer 示例
const puppeteer = require('puppeteer');
(async () => {
  const browser = await puppeteer.connect({
    browserURL: 'http://127.0.0.1:9333'
  });
  const page = await browser.newPage();
  await page.goto('https://example.com');
  console.log(await page.title());
  await browser.disconnect();
})();
```

## 七、REST API

PureCloak 内置 REST API 服务器（默认端口 `9334`），用于程序化管理工作空间。

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/status` | 获取服务器状态 |
| `GET` | `/api/workspaces` | 获取所有工作空间 |
| `GET` | `/api/workspace/<id>` | 获取单个工作空间详情 |
| `POST` | `/api/workspace` | 创建工作空间 |
| `PUT` | `/api/workspace/<id>` | 更新工作空间 |
| `DELETE` | `/api/workspace/<id>` | 删除工作空间 |
| `GET` | `/api/workspace/<id>/launch` | 启动工作空间 |
| `GET` | `/api/workspace/<id>/stop` | 停止工作空间 |
| `GET` | `/api/stats` | 获取仪表盘统计 |
| `GET` | `/api/search?q=<query>` | 搜索工作空间 |
| `POST` | `/api/proxy-test` | 测试代理连接 |

### 示例

```bash
# 获取所有工作空间
curl http://localhost:9334/api/workspaces

# 创建工作空间
curl -X POST http://localhost:9334/api/workspace \
  -H "Content-Type: application/json" \
  -d '{"name":"测试工作空间","type":"fingerprint","proxy":"socks5://127.0.0.1:1080"}'

# 启动工作空间
curl http://localhost:9334/api/workspace/<id>/launch

# 获取仪表盘统计
curl http://localhost:9334/api/stats
```

## 八、命令行参数

PureCloak 基于 Chromium，支持所有标准 Chromium 命令行参数，并增加以下 PureCloak 特有参数：

| 参数 | 说明 |
|------|------|
| `--purecloak-api-port=<port>` | REST API 服务器端口（默认 `9334`） |
| `--purecloak-cdp-port=<port>` | CDP 监听起始端口（默认 `9233`） |
| `--purecloak-data-dir=<path>` | 工作空间数据存储路径 |
| `--purecloak-max-workspaces=<n>` | 最大工作空间数量 |
| `--purecloak-disable-watchdog` | 禁用看护进程 |
| `--purecloak-log-level=<level>` | 日志级别（debug/info/warn/error） |

### 子进程参数

启动工作空间时，子进程自动附加以下参数：

- `--user-data-dir=<独立数据目录>`
- `--proxy-server=<工作空间代理>`
- `--remote-debugging-port=<独立 CDP 端口>`
- `--force-webrtc-ip-handling-policy=disable_non_proxied_udp`
- `--disable-blink-features=AutomationControlled`

## 十一、常见问题

### Q: 普通工作空间和指纹工作空间有什么区别？

**A:** 普通工作空间 = 纯净的 Chromium 浏览器，零额外隔离成本，适合日常使用。指纹工作空间 = 多账号防关联模式，每个工作空间有独立代理、独立指纹参数、独立子进程和独立存储，适合运营多个账号。

### Q: 工作空间创建的 Profile 文件存在哪里？

**A:** 默认存储在 `~/.config/purecloak/workspaces.json`（可通过 `--purecloak-data-dir` 修改）。

### Q: 如何确认反侦测生效？

**A:** 在指纹工作空间中访问 https://pixelscan.net 或 https://amiunique.org 进行检测。或者运行 `python3 tests/playwright_phase4_test.py` 查看反侦测测试结果。

### Q: 支持 Windows 或 macOS 吗？

**A:** 目前仅 Linux x86_64 支持完整功能。Windows/macOS 版本的构建脚本已就绪，但尚未深度测试。

### Q: 如何修改 CDP 端口避免冲突？

**A:** 启动时添加参数：`--purecloak-cdp-port=9333`（起始端口，每个工作空间自动递增）。
